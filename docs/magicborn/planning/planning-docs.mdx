## Section Scope

| Field | Value |
| --- | --- |
| `section` | `magicborn` |
| `source` | `apps/portfolio/content/docs/magicborn/` |
| `format` | **Planning** pages stay technical. **Lore** lives as in-world documents and archival dossiers under [`/docs/magicborn/in-world`](/docs/magicborn/in-world), grouped by **primary book**. |
| `role` | public lore/canon surface that can also feed site chat retrieval |
| `boundary` | world, characters, places, factions, magic, and canon notes live here; raw manuscript chapters do not |

## Record Rules

| Field | Rule |
| --- | --- |
| `priority` | Planning pages sort before general lore docs. |
| `shape` | Use compact tables, explicit ids, and short values instead of long prose notes. |
| `page-schema` | Prefer grouped canon dossiers: opening vignette, canon registry, known facts, tensions, and source anchors. |
| `rag-scope` | Pages in this section are retrieval-eligible unless later marked private or draft-only. |
| `manuscript-boundary` | Do not use this section as a substitute for chapter drafts under `books/`. |
| `agent-loop` | Read [requirements](/docs/magicborn/requirements), [roadmap](/docs/magicborn/planning/roadmap), [state](/docs/magicborn/planning/state), and [task registry](/docs/magicborn/planning/task-registry) first. Open this page when you need the local playbook or schema summary. |
