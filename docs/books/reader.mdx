# Book reader (EPUB)

The portfolio still builds EPUBs from the repo-root `books/` directory with `pnpm run build:books`, but the current shipped artifact no longer has to live in git. The site reads `public/books/manifest.json`, which can now point each built book at a current remote artifact URL in storage while keeping the same reader/runtime surface on the dedicated reader page and in embedded docs.

## Embedded reader in docs

The `BookReaderEmbed` component can be used in any doc or article to show the same reader used on the book reader page, loading a specific book by slug from the current manifest-backed EPUB URL.

Example:

<BookReaderEmbed slug="mordreds_tale" title="Mordred's Tale" />

## How it works

- EPUB build artifact: built from `books/<slug>/chapters/` and written locally to `apps/portfolio/public/books/<slug>/book.epub` during authoring
- Publish step: `pnpm run publish:book-artifacts` or `pnpm run build:books:publish` uploads the immutable EPUB plus a zipped planning pack to storage and refreshes the manifest with current remote URLs
- Manifest: `build:books` writes `public/books/manifest.json`, which powers the books list, `getBooks()`, and current download/reader URLs
- Runtime: the reader loads the manifest-selected URL and renders it in-browser without server-side EPUB rendering
