## EPUB build: planning supplement

The **`@portfolio/repub-builder`** CLI packs `books/<slug>/` into EPUB. Extra planning material can ride along as **appendix XHTML** in the spine (not listed in the generated table of contents):

| Mechanism | Detail |
| --- | --- |
| CLI | **`repub epub <book-folder>`** (first positional = manuscript / book source tree; there is **no** `--input` flag today) plus repeatable **`--planning <dir>`** for each planning root to embed as appendix. Example: `repub epub ./books/mordreds_tale --planning ./apps/portfolio/content/docs/books/planning --output ./out.epub`. |
| `pnpm run build:books` | Reads optional **`epubPlanningDirs`** in each `book.json` (paths **relative to that book folder**), resolves them, and passes `--planning` for each existing directory. |
| Planning pack (`pnpm dev` / `pnpm run build`) | **`build-planning-pack.cjs`** exports planning MDX from `content/docs/**` to **`/planning-pack/site/`**, including **`books/<stream>/planning/*`** (per-novel `roadmap`, `state`, `task-registry`, `decisions`, **`AGENTS.md`**) plus section `books/planning/`. Open the **Planning pack** modal on the site to download. |
| Files included | Recursive `.md`, `.mdx`, `.xml`, `.toml`, `.txt` under each root; skips `node_modules`, `.git`, `dist`, `.next`, `__pycache__`. |
| Built-in titles | Each `books/<slug>/book.json` can list `../../apps/portfolio/content/docs/books/planning` so **books** planning MDX ships inside the built EPUB (shared tree; stream-specific pages live beside section planning). Includes **Mordred's Tale**, **Mordred's Legacy**, **Magicborn: The Rune Path**. |

**Reader + Repo Planner:** the **in-browser cockpit** for a **built-in** book should load **planning pack** material from **this embedded supplement** (and eventual **repub** manifest), not from ad hoc doc URLs — see [embed architecture](/docs/documentation/reader-repo-planner-embed-architecture) (**Book-scoped planning packs**) and decisions **`BOOKS-READER-PLANNING-PACK-FROM-BOOK-ARTIFACT`**, **`BOOKS-REPUB-ARTIFACT`**. Tasks: **`books-reader-04`**.

Verify with **`pnpm run build:books`** and open the output under `apps/portfolio/public/books/<slug>/book.epub`.

**CLI productization** (help UX, **`npx` / global install**, publishing): [documentation task registry](/docs/documentation/planning/task-registry) phase **`documentation-site-07-02`**.

## Phases, task ids, and frontmatter

Shape matches [global planning](/docs/global/global-planning): **`<books>-<stream>-<phase>-<task>`** (namespace `books` is fixed for this section).

| Stream | Phases (examples) | Scope |
| --- | --- | --- |
| `reader` | `books-reader-01`, `books-reader-02` | In-app EPUB reader, annotations, read-route planning UI |
| `publishing` | `books-publishing-01` | Editions, build visibility, next live EPUB |
| `ai` | `books-ai-00`, `books-ai-01` | **`books-ai-00`** = historical first public chat on CopilotKit. **`books-ai-01`** = Payload + `sqlite-vec` RAG, citations, owner tooling, and the shipped assistant-ui public shell |

- **Phase id:** `books-<stream>-<phase>` (e.g. `books-reader-02`, `books-publishing-01`, `books-ai-01`).
- **Task id:** `books-<stream>-<phase>-<task>` (e.g. `books-ai-01-04`, `books-reader-01-16`, `books-publishing-01-02`).
- **`repoPath`:** Optional YAML frontmatter on any planning `.mdx` file: repo-relative path to this file (this page sets `repoPath: apps/portfolio/content/docs/books/planning/planning-docs.mdx`). Not shown in the article body.
- **`taskPhase`:** Optional frontmatter when a page is owned by a single phase (e.g. a future `books-ai-01-kickoff.mdx`). Omit on shared indexes like [task registry](/docs/books/planning/task-registry).

Read order for **books** work: root **`AGENTS.md`** → [requirements](/docs/books/requirements) → [roadmap](/docs/books/planning/roadmap) → [state](/docs/books/planning/state) (include [Magicborn line](/docs/books/planning/state#magicborn-line-world-order-and-chapter-beats) when writing cross-title fiction) → [task registry](/docs/books/planning/task-registry) → [decisions](/docs/books/planning/decisions) → [errors and attempts](/docs/books/planning/errors-and-attempts).

For a **single novel stream** (`mordreds-tale`, `mordreds-legacy`, `magicborn-rune-path`): that folder’s **`AGENTS.md`** (fiction loop) → **`roadmap.mdx`** → **`state.mdx`** → **`task-registry.mdx`** → **`decisions.mdx`**, then manuscript under `books/<slug>/`.
